## Answers

Take-away:
- There are a lot of components which can become very confusing when trying to create something.
- We can mix and match most of the hardware.
- https://app.cirkitdesigner.com/project for prototyping.
- Arduino IDE for coding and customizing the component interactions.
- Advice:
  - At the end of the workbook we realized that Arduino has examples for testing things in Task 9. I feel like someone should mention it in the practice session that "remember to use the provided examples, it can speed up the process when you get stuck"
  - I feel like the tasks should have more step-by-step guidance, especially in the earlier practice sessions (or more difficult tasks - task 7). Some do, but at times it's not that clear when it's a long sentence.

Good:
- The exercises had a lot of variety.
- Not too complicated, but still challenging.

Difficult:
- Most of the time we connected the dupont cables incorrectly.
- Trying to understand how to approach certain tasks, like task 7, since it had many steps. In the end we found an example image in Discord.
- Even if we do something wrong, the issue isn't clear right away, for example task 7 last point - using two Wemos D1 Mini (one for button, one for power). We had to connect both grounds with a cable, which we didn't know at first.

Fun exploration / struggling:
- Task 7 - when we finally got the lock mechanism working.
- Task 9 - when we spent a lot of time on trying to implement something and finally getting it to work (temp, touch sensor).
- Show stoppers: connecting something wrong and trying to get the code working.

Interactions:
- Interactions were good.

Help and extra work:
- We got help from instructors, they helped us out when we were struggling.
- We didn't have time to help anyone.
- No, but we did find some components (Y-cable) which were not working, so by being test subjects we helped the whole class :)
